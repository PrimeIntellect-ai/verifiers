# openenv-textarena-v1

### Overview

- **Environment ID**: `openenv-textarena-v1`
- **Short description**: Minimal OpenEnv gym adapter wrapping TextArena Wordle.
- **Tags**: openenv, gym, textarena, wordle, v1

### Datasets

- **Primary dataset(s)**: Seeded `Wordle-v0` episodes generated by TextArena.
- **Split sizes**: `num_tasks` episodes, default 100.

### Task

- **Type**: Multi-turn game with a colocated user simulator.
- **Rubric overview**: Cumulative game-authoritative reward returned by OpenEnv.

### Quickstart

```bash
uv run vf-build openenv-textarena-v1
uv pip install -e environments/openenv_textarena_v1
uv run eval openenv-textarena-v1 -n 3 --harness.runtime.type docker
```

Prime can also run the image, but its sandbox region must support port exposure
for the colocated user simulator.

### Environment Arguments

| Arg | Type | Default | Description |
| --- | --- | --- | --- |
| `num_tasks` | int | `100` | Number of seeded episodes. |
| `seed` | int | `0` | First episode seed. |
| `system_prompt` | str | Wordle prompt | Model behavior prompt. |

### Metrics

| Metric | Meaning |
| --- | --- |
| `openenv_reward` | Cumulative reward returned by OpenEnv. |

## How It Works

The taskset reads `openenv_textarena_v1/proj/.build.json`, uses its image as the
rollout image, starts the OpenEnv server on port 8001, and drives its gym
`reset`/`step` protocol through a colocated v1 `User`.

The environment-specific Python is only the thin config in
`openenv_textarena_v1/taskset.py`; the bundled `proj/` directory is a
self-contained OpenEnv project and Docker recipe.
