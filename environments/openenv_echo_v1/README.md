# openenv-echo-v1

### Overview

- **Environment ID**: `openenv-echo-v1`
- **Short description**: Minimal OpenEnv MCP tools through the reusable v1 adapter.
- **Tags**: openenv, mcp, tools, v1

### Datasets

- **Primary dataset(s)**: Seeded episodes generated by the bundled Echo server.
- **Split sizes**: `num_tasks` episodes, default 100.

### Task

- **Type**: Multi-turn tool use.
- **Rubric overview**: Sum of rewards returned by OpenEnv.

### Quickstart

```bash
uv run vf-build openenv-echo-v1
uv pip install -e environments/openenv_echo_v1
uv run eval openenv-echo-v1 -n 3 --harness.runtime.type prime
```

### Environment Arguments

| Arg | Type | Default | Description |
| --- | --- | --- | --- |
| `num_tasks` | int | `100` | Number of seeded episodes. |
| `seed` | int | `0` | First episode seed. |
| `instruction` | str | Echo instruction | Opening model instruction. |

### Metrics

| Metric | Meaning |
| --- | --- |
| `openenv_reward` | Cumulative reward returned by OpenEnv. |

## How It Works

The taskset reads `openenv_echo_v1/proj/.build.json`, uses its image as the
rollout image, starts the OpenEnv server on port 8001, and exposes the server's
MCP tools through a colocated dynamic `Toolset`.

The environment-specific Python is only the thin config in
`openenv_echo_v1/taskset.py`; the bundled `proj/` directory is a self-contained
OpenEnv project and Docker recipe.
