### Overview
- **Environment ID**: `irreps-tensor-product`
- **Short description**: Single-turn O(3) irreducible representation tensor product decompositions with exact symbolic scoring.
- **Tags**: math, representation-theory, single-turn, eval

### Datasets
- **Primary dataset(s)**: Procedurally generated by this environment
- **Source links**: local deterministic generator
- **Split sizes**: Configurable counts for train/eval via loader args

### Task
- **Type**: single-turn
- **Rubric overview**: Exact canonical symbolic match after whitespace normalization.
- **Prompt contract**: The model receives the tensor-product rule in the prompt.
- **Answer contract**: Canonical decompositions use ascending e3nn-style irreps separated by ` + `, such as `1o + 2o + 3o`.

### Quickstart
Run an evaluation with default settings:

```bash
prime eval run irreps-tensor-product
```

Configure model, sampling, and task difficulty:

```bash
prime eval run irreps-tensor-product \
  -m openai/gpt-4.1-mini \
  -n 20 -r 3 -t 1024 -T 0.7 \
  -a '{"num_train_examples": 2000, "num_eval_examples": 2000, "seed": 0, "min_l": 0, "max_l": 4}'
```

Notes:
- Use `-a` / `--env-args` to pass environment-specific configuration as a JSON object.
- The Python module name is `irreps_tensor_product`; the CLI environment ID uses `irreps-tensor-product`.
- The task is standalone and does not require `reasoning-gym`.
- e3nn notation is used for task notation only. e3nn is not a runtime dependency of this environment.
